# RLiable Results Directory

This directory contains statistical analysis results and visualizations generated by RLiable analysis scripts.

## Directory Structure

```
results/
├── plots/                      # PNG visualizations
│   ├── aggregate_metrics_reward.png
│   ├── aggregate_metrics_cost.png
│   ├── performance_profiles_reward.png
│   ├── performance_profiles_cost.png
│   ├── probability_of_improvement_reward.png
│   ├── probability_of_improvement_cost.png
│   ├── sample_efficiency_reward.png
│   └── sample_efficiency_cost.png
├── metrics/                    # JSON metric files
│   ├── aggregate_metrics_reward.json
│   ├── aggregate_metrics_cost.json
│   ├── performance_profiles_reward.json
│   ├── performance_profiles_cost.json
│   ├── probability_of_improvement_reward.json
│   ├── probability_of_improvement_cost.json
│   ├── sample_efficiency_reward.json
│   └── sample_efficiency_cost.json
└── reports/                    # Markdown reports
    ├── summary_report_reward.md
    └── summary_report_cost.md
```

## Output Types

### plots/
Contains publication-ready visualizations in PNG format.

**Aggregate Metrics Plots:**
- Bar charts showing mean, median, IQM, and optimality gap
- Separate plots for reward and cost metrics
- Error bars represent 95% confidence intervals via stratified bootstrap

**Performance Profiles:**
- CDF plots showing probability of achieving various performance thresholds
- Compares different algorithms/configurations
- Useful for understanding performance distribution across runs

**Probability of Improvement:**
- Heatmap showing probability that one algorithm outperforms another
- Statistical significance testing via bootstrap sampling
- Separate analysis for reward and cost objectives

**Sample Efficiency Curves:**
- Learning curves showing performance vs. training steps
- Shows median performance with confidence intervals
- Useful for comparing data efficiency of different approaches

### metrics/
Contains raw numerical results in JSON format for further processing.

Each JSON file contains:
- Computed statistics (means, medians, IQMs, etc.)
- Bootstrap samples for confidence interval estimation
- Algorithm/configuration identifiers
- Metadata about analysis parameters

**File naming convention:** `{metric_type}_{objective}.json`
- `metric_type`: aggregate_metrics, performance_profiles, probability_of_improvement, sample_efficiency
- `objective`: reward, cost

### reports/
Contains human-readable Markdown summaries of analysis results.

**Summary Reports:**
- Overview of best performing configurations
- Statistical comparisons between algorithms
- Key findings and insights
- Tables of quantitative results

**File naming convention:** `summary_report_{objective}.md`
- `objective`: reward, cost

## Result Metrics Explained

### Aggregate Metrics

- **Mean**: Average performance across all runs (sensitive to outliers)
- **Median**: Middle value of performance distribution (robust to outliers)
- **IQM (Interquartile Mean)**: Mean of middle 50% of runs (robust alternative to mean)
- **Optimality Gap**: Difference from optimal performance (lower is better)

### Performance Profiles

- Shows cumulative distribution function (CDF) of normalized performance
- X-axis: Performance threshold (τ)
- Y-axis: Fraction of runs achieving performance ≥ τ
- Higher curves indicate better overall performance

### Probability of Improvement

- Matrix showing P(Algorithm A > Algorithm B)
- Values > 0.5 indicate A likely outperforms B
- Bootstrap-based statistical testing with 95% confidence

### Sample Efficiency

- Performance as a function of training steps
- Shows learning speed and final performance
- Confidence intervals via bootstrap stratified sampling

## Usage

Results are automatically generated by running:

```bash
cd rliable/scripts
python compute_rliable_metrics.py
```

To analyze specific metrics:
```bash
python compute_rliable_metrics.py --metric reward_mean
```

To customize output location:
```bash
python compute_rliable_metrics.py --output-dir /path/to/custom/results
```

## Notes

- All confidence intervals use stratified bootstrap with 50,000 samples
- Performance normalization is relative to min/max observed values per task
- Results are organized by objective (reward vs cost) for easy comparison
- JSON files can be loaded for custom plotting or further statistical analysis
- Plots use seaborn styling for publication-quality figures

## Interpreting Results

### For Reward Maximization:
- Higher values are better
- Look for algorithms with high median/IQM in aggregate metrics
- Performance profiles should be high (right-shifted)

### For Cost Minimization:
- Lower values are better
- Look for algorithms with low median/IQM in aggregate metrics
- Performance profiles interpretation is inverted (still higher curves better after normalization)

### Statistical Significance:
- Use probability of improvement matrices to assess pairwise differences
- Values significantly above/below 0.5 indicate meaningful differences
- Bootstrap confidence intervals provide uncertainty quantification
